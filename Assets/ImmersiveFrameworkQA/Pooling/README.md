# Pooling QA Harness

This folder contains the QA-only harness for validating `com.immersive.pooling` as an independent package consumer.

## Create the scene

Use Unity menu:

```text
Immersive Framework QA > Pooling > Create or Refresh Pooling QA Scene
```

The editor tool creates or refreshes:

```text
Assets/ImmersiveFrameworkQA/Pooling/Scenes/QA_Pooling.unity
Assets/ImmersiveFrameworkQA/Pooling/Prefabs/QA_PooledCube.prefab
Assets/ImmersiveFrameworkQA/Pooling/Prefabs/QA_PooledSphere.prefab
Assets/ImmersiveFrameworkQA/Pooling/ScriptableObjects/QA_CubePool.asset
Assets/ImmersiveFrameworkQA/Pooling/ScriptableObjects/QA_LimitedCubePool.asset
Assets/ImmersiveFrameworkQA/Pooling/ScriptableObjects/QA_AutoReturnPool.asset
```

The scene is generated by Unity to avoid hand-authored YAML.

## Manual smoke

1. Open `QA_Pooling.unity`.
2. Press Play.
3. Use the IMGUI panel.
4. Run:
   - `Run Basic Smoke`
   - `Run Max Limit Smoke`
   - `Run Auto Return Smoke`
5. Optionally use manual buttons:
   - `Prewarm`
   - `Rent One`
   - `Rent Three`
   - `Return Last`
   - `Return All`
   - `Clear Active`

## Expected results

Basic smoke should prove:

```text
Prewarm -> Rent 3 -> Return 1 -> Rent 1 -> Return All
```

Max limit smoke should prove:

```text
maxSize=2 and canExpand=false blocks the third rent safely.
```

Auto return smoke should prove:

```text
autoReturnSeconds returns the rented object automatically.
```

The panel displays snapshots and callback counters. Logs use `[POOLING_QA]`.

## Boundary

This QA harness may depend on `com.immersive.pooling`.
It must not depend on `com.immersive.framework`, Audio, FIRSTGAME, or GameJam2025 sources.
