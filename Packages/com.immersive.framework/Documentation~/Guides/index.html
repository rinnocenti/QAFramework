Guia F0B — Baseline Hygiene

Este guia foi reduzido para refletir o começo correto do roadmap. F0B aplica a higiene mínima depois dos ADRs de F0A; o próximo corte técnico é F1.

Estado atual

Fase aceita: F0B — Baseline hygiene.

Próximo corte técnico: F1 — API status, Identity and Diagnostics.

O framework ainda não deve avançar para features. A prioridade seguinte é dar status, identidade e diagnostics mínimos para impedir novas APIs ambíguas.

Este guia não promete execução de subsistemas deferred, como additive scenes. RouteContentRuntime está ativo na F3 apenas para callbacks locais de Route Content na Primary Scene carregada. CameraFlow foi removido do core ativo.

Como validar F0B

  1. Confirmar que o projeto compila sem erros CS.
  2. Confirmar que com.immersive.framework não depende mais de com.unity.cinemachine.
  3. Confirmar que Runtime/CameraFlow não faz parte do core ativo.
  4. Executar o smoke mínimo descrito em Documentation~/BASELINE_SMOKE.md.
