Estado atual

Fase atual: F6 — CLOSED / ROUTE SCENE COMPOSITION + RELEASE BASELINE PASS.

O framework resolve Game Application, inicia Route/Activity, executa callbacks locais, descobre local contributions e agora compõe cenas de Route por RouteContentProfileAsset.

Uma Route pode carregar uma Primary Scene em Single e additional scenes owned em Additive. Ao sair da Route, additional scenes owned são descarregadas pelo release plan.
F6 não cria Surface, RuntimeRoot, prefab materialization, Actor, Input, Camera, Save, Reset, Pooling ou Activity canonical materialization.

Route Content Profile

  1. Selecione o asset Route.
  2. No Inspector, atribua ou crie um Content Profile.
  3. No RouteContentProfileAsset, adicione uma entrada em Additional Route Scenes.
  4. Use um Content Id explícito e estável.
  5. Atribua uma cena válida e garanta que ela esteja no Build Profile ou Shared Scene List.
  6. Use Required para bloquear a Route se a cena falhar, ou Optional para apenas reportar issue.
  7. Mantenha ownership como Owned quando o framework deve descarregar a cena no Route exit.

Não coloque outro bootstrap do framework dentro da additional scene.

Logs esperados

Com uma Primary Scene e uma additional scene owned:

routeSceneComposition='Succeeded'
routeSceneLoaded='2'
routeSceneFailed='0'
routeSceneBlockingIssues='0'
routeContentHandles='2'

Ao trocar para outra Route:

routeRelease='Succeeded'
routeReleaseReleased='1'
routeReleaseSkipped='1'
routeReleaseFailed='0'
routeReleaseBlockingIssues='0'

Released='1' é a additional scene owned descarregada. Skipped='1' é a Primary Scene ativa, que continua controlada por LoadSceneMode.Single.
