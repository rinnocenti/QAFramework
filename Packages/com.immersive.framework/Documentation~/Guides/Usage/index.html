Immersive Framework

Guia de Uso Inicial

Guia incremental do package com.immersive.framework. Este documento registra apenas o que existe no corte atual e deve crescer junto com o framework.

Estado atual

Corte atual: IF-FW-2S - Activity Lifecycle Events.

O framework agora emite eventos canônicos de lifecycle de Activity por meio de Foundation.Events, mantendo o mesmo comportamento visual de Activity Content Binding.

O registro de Activity continua usando FrameworkLogger e não ganhou service locator, event bus global ou refatoração de consumo de eventos no conteúdo de cena.

O framework já consegue resolver uma aplicação ativa, validar uma Startup Route, validar a Primary Scene, criar um runtime persistente, iniciar uma Route por Route Lifecycle, trocar Route via Route Request Trigger, iniciar/trocar/limpar Activity via Activity Request Trigger e aplicar conteúdo authored de Activity.

O bootstrap cria um objeto interno Immersive Framework Runtime marcado como DontDestroyOnLoad. Esse objeto é o owner de escopo de aplicação e mantém o Game Flow vivo durante cargas Single. O Game Flow delega o início concreto da rota para Route Lifecycle, inclusive em solicitações runtime.
Para testar uma cena isolada sem boot do framework, use Project Settings > Immersive Framework > Editor Play Mode Startup > Current Scene Only.

Conceitos existentes

Game Application

Asset raiz do jogo. Define identidade da aplicação, Startup Route e Validation Mode.

Immersive Framework Settings

Project Settings que aponta para o Game Application ativo.

Route

Asset público que identifica uma entrada do fluxo do jogo e declara uma Primary Scene.

Activity

Asset público que identifica uma etapa de gameplay dentro de uma Route. Neste corte, a Activity tem apenas identidade e pode ser iniciada como Startup Activity, solicitada por Activity Request Trigger ou limpa por request explícito.

Game Flow

Owner runtime mínimo que aceita a Startup Route após o bootstrap e delega o início concreto da rota para o Route Lifecycle.

Route Lifecycle

Owner runtime mínimo que inicia uma Route e delega o carregamento/ativação de cena para o Scene Lifecycle. Ele existe para impedir que Game Flow ou Bootstrap virem owners de cena.

Application Runtime

Owner runtime persistente criado pelo bootstrap. Mantém o Game Flow deste boot e guarda um estado mínimo de diagnóstico. Não é service locator nem asset de configuração.

Route Request Trigger

Componente público para objetos de cena ou botões de UI solicitarem outra Route sem expor pipeline, stage, command, fact ou snapshot.

Activity Request Trigger

Componente público para objetos de cena ou botões de UI solicitarem outra Activity dentro da Route ativa.

Activity Content Binding

Marcador simples para GameObjects de cena que devem ficar ativos apenas enquanto uma Activity específica está ativa.

Framework Logger

Fronteira interna de logging do framework. Compoe com.immersive.logging sem criar logger global, service locator ou UI de diagnostics.

Primeira configuração

  1. Abra Project Settings > Immersive Framework.
  2. Clique em Create and Assign Game Application.
  3. Selecione o asset Game Application.
  4. Na seção Startup, clique em Create and Assign Startup Route.
  5. Selecione o asset Route criado.
  6. Na seção Scene, atribua uma Primary Scene.
  7. Opcionalmente atribua uma Startup Activity na Route.
  8. Opcionalmente adicione Activity Content Binding em GameObjects de cena.
  9. Entre em Play Mode.

Também é possível selecionar um Game Application e usar Set as Active Game Application no Inspector.

Logs esperados

Com Active Game Application, Startup Route e Primary Scene configurados, o boot deve passar.

Quando a Startup Scene já está aberta:

Game Flow started with Startup Route 'Startup Route'. Route Lifecycle started Route 'Startup Route'. Scene Lifecycle resolved Primary Scene 'StartupScene' and set it active. alreadyLoaded='True'. loadMode='AlreadyLoaded'. Validation Mode: Standard.

Quando outra cena está aberta e a Startup Scene é carregada pelo framework:

[Immersive Framework] Boot succeeded. Application Runtime started. Game Application 'Game Application' resolved. Startup Route 'Startup Route' resolved. Primary Scene 'StartupScene' declared. Game Flow started with Startup Route 'Startup Route'. Route Lifecycle started Route 'Startup Route'. Scene Lifecycle resolved Primary Scene 'StartupScene' and set it active. alreadyLoaded='False'. loadMode='Single'. Validation Mode: Standard.

Conteúdo de Activity

Use Activity Content Binding para conteúdo simples authored diretamente na cena.

  1. Crie ou configure um asset Activity.
  2. Selecione o GameObject de cena que deve pertencer a essa Activity.
  3. Adicione Immersive Framework > Activity Content Binding.
  4. No campo Activity, atribua a Activity dona desse conteúdo.

Quando a Activity ativa muda, o framework aplica os bindings carregados na cena:

  • binding da Activity ativa: GameObject ativo;
  • binding de outra Activity: GameObject inativo;
  • sem Activity ativa: todos os bindings ficam inativos;
  • binding sem Activity atribuída: warning explícito.

Resumo esperado:

[Immersive Framework] Activity Content applied 2 binding(s) for Activity 'Activity 02'. activated='1' deactivated='1' unchanged='0'.

Detalhe esperado:

[Immersive Framework] Activity Content Binding diagnostics. activeActivity='Activity 02' observations=[object='Panel_Activity02' scene='StartupScene' assignedActivity='Activity 02' action='Activate' reason='MatchedActiveActivity'; object='Panel_Activity01' scene='StartupScene' assignedActivity='Activity 01' action='Deactivate' reason='DifferentActivity'].

Warning esperado para binding incompleto:

[Immersive Framework] Activity Content Binding warning. warnings=[object='Panel_MissingActivity' scene='StartupScene' reason='MissingActivityReference'].
Este binding é uma solução mínima para conteúdo simples de cena. Ele não substitui futuros sistemas de Actor, Spawn, Pooling, Presentation, Input, Camera ou Save. Evite aninhar bindings de Activities diferentes até existir policy própria.

Solicitar outra Activity

  1. Crie ou configure outro asset Activity.
  2. Adicione Activity Request Trigger a um GameObject da cena atual.
  3. No campo Target Activity, atribua a Activity de destino.
  4. Opcionalmente preencha Reason.
  5. Em um botão/UI Event, chame ActivityRequestTrigger.RequestActivity ou ActivityRequestTrigger.ClearActivity.

Log esperado em sucesso:

[Immersive Framework] Activity Request completed. source='ActivityRequestTrigger' reason='Target Activity'. Activity Flow switched from Activity 'Previous Activity' to Activity 'Target Activity'.

Para limpar a Activity atual sem trocar Route, conecte o botão/evento em ActivityRequestTrigger.ClearActivity.

[Immersive Framework] Activity Request completed. source='ActivityRequestTrigger' reason='Clear Activity'. Activity Flow cleared Activity 'Previous Activity' by request.

Se a Activity solicitada já estiver ativa, o request é ignorado explicitamente.

Solicitar outra Route

  1. Crie ou configure outro asset Route.
  2. Atribua uma Primary Scene para essa Route.
  3. Garanta que a cena esteja no Build Profile se ela não estiver carregada.
  4. Adicione Route Request Trigger a um GameObject da cena atual.
  5. No campo Target Route, atribua a Route de destino.
  6. Em um botão/UI Event, chame RouteRequestTrigger.RequestRoute.

Log esperado em sucesso:

[Immersive Framework] Route Request completed. source='RouteRequestTrigger' reason='Target Route'. Route Lifecycle started Route 'Target Route'. Scene Lifecycle resolved Primary Scene 'TargetScene' and set it active. alreadyLoaded='False'. loadMode='Single'.

Validação obrigatória

Configuração ausenteResultado esperado
Active Game Application Boot falha explicitamente. Não existe fallback.
Startup Route Boot falha explicitamente. A Route deve ser atribuída no Game Application.
Startup Route Primary Scene Boot falha explicitamente. A cena deve ser declarada no asset Route.

O que ainda não existe

Estes itens não fazem parte do corte atual:

  • transições entre Routes;
  • cenas adicionais por Route;
  • loading screen/transições visuais;
  • Activity content avançado, nested content policy e content loading;
  • Actor lifecycle;
  • Input, Camera, Save;
  • integração de Pooling;
  • module graph completo;
  • diagnostics avançado.

Editor Play Mode Startup

Use Project Settings > Immersive Framework > Editor Play Mode to choose how Play Mode starts while developing in the Unity Editor.

ModeBehavior
Framework StartupRuns the normal framework boot: Game Application, Startup Route, Game Flow, and Scene Lifecycle. If the Startup Scene is not already loaded, it is loaded with LoadSceneMode.Single.
Current Scene OnlySkips framework boot in the Editor so the currently open scene can be tested in isolation.

Current Scene Only is editor-only. Player builds always use framework startup.

Expected skip log:

[Immersive Framework] Boot skipped. Editor Play Mode Startup is set to Current Scene Only.

IF-FW-2I — Route switch ownership

Route Lifecycle now owns the active Route identity.

Game Flow accepts route requests and delegates route startup/switching to Route Lifecycle. It does not own the current Route state.

Route Request Trigger
  -> Application Runtime
      -> Game Flow
          -> Route Lifecycle
              -> Scene Lifecycle

Expected route switch diagnostics include the previous Route and the target Route:

[Immersive Framework] Route Request completed. source='RouteRequestTrigger' reason='Second Route'. Route Lifecycle switched from Route 'Startup Route' to Route 'Second Route'. Scene Lifecycle resolved Primary Scene 'SecondScene' and set it active. alreadyLoaded='False'. loadMode='Single'.

IF-FW-2J — Route request diagnostics ownership

Route request completion logs are now emitted by the persistent FrameworkRuntimeHost, not by the scene-authored RouteRequestTrigger.

This keeps diagnostics alive when a request loads another scene with LoadSceneMode.Single and destroys the requesting scene object.

[Immersive Framework] Route Request completed. source='RouteRequestTrigger' reason='Second Route'. Route Lifecycle switched from Route 'Startup Route' to Route 'Second Route'. Scene Lifecycle resolved Primary Scene 'SecondScene' and set it active. alreadyLoaded='False'. loadMode='Single'.

Activity Content Binding authoring guardrails

The Activity Content Binding Inspector now explains exactly what the component does: it activates this GameObject when the assigned Activity is active and deactivates it otherwise.

  • Missing Activity is shown as an authoring error. Runtime still skips it and emits a warning.
  • Nested Activity Content Bindings are shown as warnings because nested content policy does not exist yet.
  • The component does not spawn, pool, reset, save, bind actors, bind input, bind camera, or load content.

IF-FW-2T — Activity Content e eventos de Activity

O conteúdo de Activity continua usando Activity Content Binding, mas agora a aplicação de visibilidade é acionada pelo fluxo de eventos canônicos de Activity emitidos pelo ActivityFlowRuntime.

O comportamento visual não muda: a Activity ativa liga seus bindings e desliga os demais. A diferença é arquitetural: o caminho passa por ActivityEnteredEvent e ActivityExitedEvent usando Foundation.Events.

IF-FW-2U — Route Lifecycle Events

RouteLifecycleRuntime now publishes canonical route lifecycle events using com.immersive.foundation.

  • RouteEnteredEvent is emitted when a Route becomes active.
  • RouteExitedEvent is emitted when the previous Route is replaced by another Route.

This does not change visible route behavior. It creates the event boundary that future modules can consume without being called directly by Game Flow.

Framework QA Canvas

Para smokes manuais, adicione o componente Immersive Framework > QA > Framework QA Canvas em um GameObject de desenvolvimento.

Ele permite disparar Route Request, Activity Request e Clear Activity por botões IMGUI, sem criar dependência runtime com UGUI.

Use apenas como ferramenta de desenvolvimento. O componente pode persistir entre cenas para continuar disponível depois de um carregamento Single.
